# Content Management System

This directory contains all content for TC Heiner's portfolio website, organized into type-safe collections with automatic validation.

## üìÅ Content Collections

### Posts (`/posts/`)
**Blog posts and articles**
- **Naming**: `post-YYYY-MM-DD.mdx`
- **Schema**: `title`, `startDate`, `description?`, `image?`, `tags?`, `canonical?`
- **Indexed by AI**: ‚úÖ (Full content searchable by chatbot)

### Projects (`/projects/`)  
**Portfolio project showcases**
- **Naming**: `project-YYYY-MM-DD.mdx` or descriptive names
- **Schema**: `title`, `startDate`, `description`, `image?`, `tags?`
- **Indexed by AI**: ‚úÖ (Technical details and accomplishments)

### Experiences (`/experiences/`)
**Professional work experience**
- **Naming**: `experience-YYYY-MM-DD.mdx` or company names
- **Schema**: `title`, `company`, `startDate`, `endDate?`, `tags?`
- **Indexed by AI**: ‚úÖ (Skills, responsibilities, achievements)
- **Current positions**: Omit `endDate` field

### Books (`/books/`)
**Book reviews and reading notes**
- **Naming**: `book-YYYY-MM-DD.mdx` or book titles
- **Schema**: `title`, `author`, `readYear`, `tags?`
- **Indexed by AI**: ‚ùå (Personal reading, not professional content)

### Recipes (`/recipes/`)
**Cooking recipes and culinary experiments**  
- **Naming**: `recipe-YYYY-MM-DD.mdx` or dish names
- **Schema**: Complex recipe schema (see template)
- **Indexed by AI**: ‚ùå (Personal content, not professional)


### Manual Creation
1. Copy appropriate template from `/templates/`
2. Rename with proper naming convention
3. Fill in frontmatter and content
4. Validate with `node validate-mdx.js`

### Content Guidelines

#### Frontmatter Requirements
- **All dates**: Use `YYYY-MM-DD` format
- **Tags**: Array of strings `["tag1", "tag2"]`
- **Images**: Relative paths from `/public/` directory
- **Optional fields**: Include `?` in schema, can be omitted

#### Content Best Practices
- **Professional content** (posts, projects, experiences) should be detailed for AI indexing
- **Use descriptive titles** and tags for better discoverability
- **Include images** when available for visual appeal
- **Link to related content** using relative paths

### Common Schema Errors
```yaml
# ‚ùå Wrong date format
startDate: 2025/09/02      # Wrong
startDate: 2025-09-02      # Correct

# ‚ùå Invalid tags format  
tags: "tag1,tag2"          # Wrong - string
tags: ["tag1", "tag2"]     # Correct - array

# ‚ùå Missing required fields
# title: "Missing title"   # Wrong - commented out
title: "My Project"        # Correct

# ‚ùå Current position with end date
endDate: null              # Wrong - use null or omit
# endDate:                 # Correct - commented out or omitted
```

## ü§ñ AI Chatbot Integration

### What Gets Indexed
The AI chatbot automatically indexes content for intelligent Q&A:

**Indexed Collections** (searchable by users):
- ‚úÖ **Posts**: Full blog post content, code examples, insights
- ‚úÖ **Projects**: Technical details, challenges, solutions, tech stack
- ‚úÖ **Experiences**: Job responsibilities, achievements, skills used

**Not Indexed** (personal content):
- ‚ùå **Books**: Reading notes and personal reflections
- ‚ùå **Recipes**: Cooking content not relevant for professional queries

### Content Update Impact
When you modify indexed content (posts/projects/experiences):

1. **FAISS index rebuild required** for AI to see changes
2. **Container redeployment needed** for AWS Lambda updates
3. **Responses improve** with fresh, accurate information

```bash
# After editing indexed content:
cd backend
python -c "from chatbot.services import rebuild_vectorstore; rebuild_vectorstore()"
./build-container.sh  # Deploy to AWS
```
